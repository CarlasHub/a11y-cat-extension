A11Y Cat

Local-first accessibility inspection extension

A11Y Cat helps developers, QA teams, and accessibility reviewers inspect live pages with clear separation between confirmed findings, review-required items, advisory notes, and explicit scan limitations.

Private beta
Read the docs View GitHub repository
Privacy boundary: No developer server, no developer database, local processing in the browser runtime.

Feature overview

axe-core scan results

Runs bundled axe-core and preserves engine provenance in grouped output.

Confirmed issues

Shows high-confidence deterministic findings separately from review-only results.

Manual/review items

Keeps incomplete, visual, contextual, and state-limited items in review buckets.

Severity filters

Lets teams prioritize Critical, Serious, Moderate, Minor, and review counts.

Exports

Provides CSV issue logs, JSON evidence bundles, diagnostics, and local issue-state exports.

Local history

Stores scoped scan history locally for previous-scan comparison by origin and pathname.

Screen Reader Review aid

Supports structured virtual review workflows. It is not real VoiceOver, NVDA, or JAWS testing.

Metadata and broken links

Surfaces metadata quality and link verification outcomes with explicit limitations.

Language and spelling review

Flags language mismatch and spelling concerns as review/advisory signals, not automatic WCAG failures.

Diagnostics and limitations

Reports runtime constraints and partial-inspection boundaries for technical troubleshooting.

Scanner clarity

Use this matrix to interpret results before creating defects or compliance statements.

Result classification model
Category Meaning Confidence Primary action
Confirmed findings Automated findings with deterministic confidence from axe violations and deterministic checks. Highest automated confidence Fix first by severity and retest.
Review items Incomplete, visual, context-dependent, or state-limited results needing human judgement. Manual verification required Inspect manually before marking as defect.
Advisory notes Quality guidance such as metadata, spelling, or language refinement signals. Guidance only Use for quality improvement, not automatic fail claims.
Limitations Runtime or context boundaries where the scanner could not fully inspect the page. Coverage boundary signal Run manual checks in missing contexts.

What this tool does not prove

  • It does not prove full WCAG compliance.
  • It does not replace manual accessibility testing.
  • It does not replace real assistive-technology testing with VoiceOver, NVDA, or JAWS.
  • Private beta limitations remain and must be treated as release constraints.
