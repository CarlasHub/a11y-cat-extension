Guide overview

Extension UI feature guide

This overview explains how to read results and where each feature family is documented in focused child pages.

What this guide covers

The full UI guide is split into focused pages so each workflow is readable on desktop and mobile. All visible extension feature families are documented across the pages below.

How to read results

  1. Start with Confirmed Issues.
  2. Prioritize by Critical, Serious, Moderate, Minor.
  3. Treat Review and Advisory items as human-validation workflow, not automatic defects.
  4. Read Scan limitations and diagnostics before deciding coverage is complete.

Result classification summary

Result classification quick model
CategoryMeaningNeeds human reviewPrimary action
ConfirmedHighest automated confidence in current state.No, but verify context before sign-off.Fix and retest.
ReviewAutomation could not safely decide pass/fail.Yes.Manual validation.
AdvisoryGuidance signal, not an automatic failure.Usually yes.Policy/team judgement.
LimitationCoverage/runtime boundary.Yes.Test missing states/contexts manually.
Boundary: A11Y Cat supports review workflows. It does not prove WCAG conformance and does not replace manual accessibility testing or real assistive-technology validation.
