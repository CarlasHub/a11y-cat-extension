Result classification matrix

This matrix gives a fast interpretation model for counts, labels, and next actions before reading individual issue cards.

Classification and action model used in the extension UI
UI bucket Primary data sources How count is interpreted WCAG relationship User action
Confirmed Issues axe violations and deterministic module output Confirmed findings count in the tested page state Direct when mapping exists in finding metadata Prioritize by severity, fix, rerun, compare
Manual / Incomplete / Review Items axe incomplete and review modules Review candidates, not auto-confirmed failures Depends on manual outcome and context Manually inspect before defect filing
Advisory Notes spelling, language, metadata, guidance checks Quality guidance signal count Usually indirect or advisory-only Use as quality backlog, not hard fail list
Scan Limitations diagnostics and runtime boundary records Coverage limitations count, not pass/fail count Not a failure category Investigate missing coverage manually
