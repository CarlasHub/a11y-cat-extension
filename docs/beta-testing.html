Beta testing guide

This extension is documented for controlled pilot evaluation on desktop Chromium. Test it as a bounded review tool, not as a production compliance certificate.

How to install the beta

  1. Build or obtain the current beta package.
  2. Open chrome://extensions.
  3. Enable Developer mode.
  4. Use Load unpacked for a local build or install the agreed beta package.

How to test

  1. Open a supported Chrome / Chromium desktop page.
  2. Run the deterministic WCAG scan from the toolbar panel.
  3. Review Confirmed Issues first.
  4. Review review buckets and scan limitations next.
  5. Use exports for handoff only after checking sensitive snippets and local notes.
  6. Verify real AT behavior separately with manual VoiceOver, NVDA, or JAWS testing where required.

How to report bugs

Report beta bugs through the public repository issue tracker:

Open the issue tracker

What evidence to provide

  • page URL and page title, if you are allowed to share them
  • what you clicked and what state the page was in
  • expected result and actual result
  • whether the issue was confirmed, review-only, advisory, or a limitation
  • diagnostics export, if runtime boundaries are involved
  • screenshots or exported evidence, if allowed
  • browser version and extension version/build identifier

What not to scan without permission

  • private customer environments you are not authorized to access
  • production systems with confidential content unless your role allows it
  • authenticated or internal pages that your testing scope does not cover
  • third-party sites where automated requests or inspection would violate policy
