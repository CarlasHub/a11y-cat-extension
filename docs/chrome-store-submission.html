Detailed description

A11Y Cat is a Chrome Manifest V3 extension that supports accessibility review of the current browser tab. After the user clicks the toolbar action, it injects packaged local scripts into the active tab, runs bundled axe-core checks, runs A11Y Cat review modules, separates confirmed findings from review/advisory/diagnostic output, and lets the user export local evidence for remediation and QA handoff.

The extension does not replace manual accessibility testing, real assistive-technology validation, keyboard-only testing, responsive/user-task review, or legal accessibility sign-off.

Permissions justification

PermissionReason
activeTabTemporary current-tab access after toolbar click and user-triggered visible-tab screenshot capture where Chrome permits it.
scriptingInjects packaged local runtime scripts into the active tab.
storageSaves local settings, scan history, review state, and allowlists on the browser profile.
debuggerApplies and restores temporary Chrome mobile viewport emulation for desktop/mobile comparison.

Data usage disclosure

  • Scan processing runs locally in the extension runtime.
  • The public package does not declare analytics, telemetry, hosted scanner APIs, or remote scanner code.
  • Exports are user-triggered and may contain sensitive page-derived evidence.
  • Same-origin broken-link checks can make page-context requests to verify links. Cross-origin links are not fetched automatically.

Version and release notes

  • Version 1.0.0 beta focuses on toolbar-launched current-page accessibility review, evidence classification, local scan history, Screen Reader Review support, Browser Preference QA Controls, and user-triggered exports.
  • The active extension target is Chrome or Chromium desktop. Firefox and Safari are not supported release targets.
  • Documentation is prepared for store review, but final upload still requires current release verification, package inspection, final screenshots, and maintainer support email configuration.

Screenshots and assets checklist

  • Show the extension toolbar-launched panel on a safe test page.
  • Show scan results, dashboard evidence, scanner modules, Browser Preference QA Controls, Screen Reader Review, privacy/local-data disclosure, and limitations without private client data.
  • Use current extension UI screenshots only; do not use legacy bookmarklet images or internal QA evidence bundles.
  • Confirm icon assets and screenshots contain no secrets, private URLs, or unpublished implementation notes.

Package and upload checks

Before upload, run npm run verify:store-submission, inspect the zip generated by npm run package:store-submission, and confirm it contains only required runtime files. Do not upload tests, Playwright reports, traces, coverage, screenshots/videos, local QA artefacts, private prompts, source handoff bundles, or assistant backend endpoints.

Submission boundary

This page prepares store documentation. Chrome Web Store upload readiness still requires the latest successful release verification, package inspection, final screenshots, and a maintainer-configured support email.
