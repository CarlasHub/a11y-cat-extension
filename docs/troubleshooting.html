Troubleshooting

Common extension error states, likely causes, and direct fixes.

Extension panel does not open

  • Cause: restricted page context such as chrome://, chrome-extension://, or blocked file://. Fix: open a standard https:// page or enable file access for the extension.
  • Cause: extension disabled. Fix: enable A11Y Cat in chrome://extensions, then reload the page.
  • Cause: injection blocked by page runtime constraints. Fix: refresh and retry, then inspect Diagnostics if the issue persists.

Scan produces no results

  • Cause: runtime was not injected. Fix: close and reopen the panel, then rerun scan.
  • Cause: page loaded before extension action. Fix: refresh after opening the panel.
  • Cause: restricted/protected context. Fix: test in a supported page context and check limitations output.

Scan appears stuck

  • Cause: page is still mutating or loading heavy client-side content. Fix: wait for page idle, then rerun scan.
  • Cause: protected frame/content boundary reduced inspectable scope. Fix: inspect scan limitations and diagnostics output.
  • Cause: runtime fault in current tab. Fix: reload tab, reopen panel, rerun scan, and capture support diagnostics if repeated.

Restricted page contexts

  • Chrome Web Store pages: extension injection is restricted; these pages cannot be scanned as normal page targets.
  • chrome:// pages: browser-internal pages cannot be scanned by this extension runtime.
  • chrome-extension:// pages: extension pages are outside the normal scan target scope.

file:// pages cannot be scanned or require file access

  • Cause: extension file URL access is disabled. Fix: in chrome://extensions, open A11Y Cat details and enable file URL access.
  • Cause: local file context still has runtime limits. Fix: treat output as potentially partial and validate manually.

History or comparison is missing

  • Cause: first scan in this origin + pathname scope. Fix: run another scan on the same scope.
  • Cause: history cleared. Fix: rerun scans to repopulate local history.
  • Cause: scope changed (different origin/pathname). Fix: compare from the same scoped page; query/hash are ignored.

Export file is empty or missing fields

  • Cause: export triggered before running a scan. Fix: run scan first, then export.
  • Cause: partial scan from restricted context. Fix: inspect scan limitations and diagnostics output.

Spelling check shows no results

  • Cause: page language unsupported by bundled dictionary. Fix: treat as known limitation and use manual review.
  • Cause: missing/unknown lang context. Fix: set language metadata and rescan.
  • Cause: no visible text nodes in scanned state. Fix: test state where text is rendered.

See Known Limitations for language and state boundaries.

Broken Links shows unverified for all links

Cross-origin links are not fetched by design and stay unverified. Same-origin links can be checked with HEAD/GET requests from page context.

Metadata or schema output is missing/invalid

  • Cause: page does not include expected Open Graph/Schema.org/JSON-LD markup. Fix: treat as advisory/quality feedback and verify page requirements.
  • Cause: JSON-LD block exists but is invalid JSON. Fix: validate syntax and rerun scan.
  • Cause: dynamic metadata is not present in current state. Fix: test the state where metadata is emitted.

Screen Reader Review is not real AT proof

Screen Reader Review is a virtual review aid and workflow helper. It is not evidence of VoiceOver, NVDA, or JAWS behavior. Use real assistive-technology testing for accessibility sign-off.

How to clear local data

  • Open the extension panel and use Scan Results → Privacy and data use → Clear all local extension data.
  • You can also remove the extension to clear extension-managed local storage.

Need support diagnostics for issue reports

  • Use diagnostics export/copy from the extension panel.
  • Attach diagnostics with reproduction steps in your bug report.
