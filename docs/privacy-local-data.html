Privacy and local data

The public extension processes page content locally in the browser.

  • No developer database is used by the public release package.
  • No developer server is required by the public release package.
  • No analytics or telemetry endpoint is included in the public release package.

What is stored locally

  • Scan analysis results needed for local history and comparison
  • Issue annotations and local workflow state
  • Saved settings and panel geometry
  • Saved scan history and previous-scan comparison state
  • Saved Screen Reader Review state
  • Local benchmark history
  • Saved spelling allowlist entries

Current local storage keys

  • a11yCatExtSettingsV1
  • a11yCatScanHistoryV1
  • a11yCatScanHistoryArchiveV1
  • a11yCatWorkflowStateV1
  • a11yCatVirtualSrReviewStateV1
  • a11yCatPerfBenchmarksV1
  • a11yCatSpellingAllowlistV1

What exports may include

  • page URL and sometimes page title
  • selectors and evidence snippets
  • rule ids, WCAG references, and classification fields
  • local notes and workflow state
  • diagnostics and limitation records
  • virtual Screen Reader Review logs and QA notes where those exports are used

Request behavior and limits

  • The broken-links check can make same-origin HEAD or GET requests to verify linked destinations.
  • Cross-origin links are not fetched automatically; they remain unverified.
  • Metadata checks can trigger normal image requests for declared preview-image URLs when the page itself would ordinarily request them.

How users clear data

  • Scan Results → Privacy and data use → Clear all local extension data
  • Removing the extension also clears extension-managed storage.

Limitations

  • Local-only processing does not mean every page can be fully inspected.
  • Restricted pages, protected frames, and unsupported contexts can still limit coverage.
  • Exports are local-only until the user chooses to copy or share them.
