Frequently asked questions

Direct answers for developers, QA teams, accessibility reviewers, and stakeholders using A11Y Cat in private beta workflows.

Does A11Y Cat prove WCAG compliance?

No. A11Y Cat supports accessibility review workflows and triage. It does not prove full compliance. Review findings in context and complete manual accessibility testing and real assistive-technology validation.

Does A11Y Cat replace manual testing?

No. Automated scanning reduces triage time, but manual accessibility testing is still required for interaction flow, context, and assistive-technology behavior.

Does A11Y Cat replace screen reader testing?

No. The extension’s Screen Reader Review is a virtual review aid, not real VoiceOver, NVDA, or JAWS validation evidence.

What WCAG version does A11Y Cat target?

The bundled axe-core configuration runs WCAG A/AA tags for WCAG 2.0, 2.1, and 2.2 where available. A11Y Cat preserves WCAG mappings in finding metadata, but does not provide conformance certification.

Is Screen Reader Review the same as testing with a real screen reader?

No. Screen Reader Review is a virtual review aid. It is not VoiceOver, NVDA, or JAWS evidence. Real assistive-technology testing is still required for sign-off.

What does "corroborated failure" mean?

A corroborated failure is a confirmed failure where at least two independent detection mechanisms flag the same element for the same reason within one scan. Example: axe-core and the custom alt-text analyzer both flag the same image for missing meaningful text.

Does the extension send data to a server?

The public release build processes data locally. It does not automatically send scan data to a developer server or database. Exports leave the device only when users choose to share them.

Does A11Y Cat use analytics?

No analytics or telemetry endpoint is included in the public beta package documentation scope.

What does local-first mean?

Local-first means scan processing, classification, and storage are handled in the local browser extension runtime. Data only leaves the machine when a user chooses to export or share output.

Why are some items review or advisory instead of confirmed failures?

Some checks depend on context, state, visual interpretation, or incomplete runtime evidence. Those findings are classified as review/advisory to prevent false certainty.

What is Visual Composition Review?

Visual Composition Review groups items that may require visual judgment (for example layout relationships and visual meaning) and are not automatically confirmed failures.

Can I use this on internal or authenticated pages?

You can only test pages you are authorized to inspect. Restricted contexts, protected frames, and state boundaries may limit what the scanner can inspect.

Can I use this in Microsoft Edge?

Edge (Chromium) is supported in limited beta validation. Chrome desktop is the primary supported runtime in this release.

What does a review item mean?

A review item means automation detected a potential issue that requires manual judgment. Do not treat review items as confirmed defects until verified.

What does Previous Scan compare?

Previous Scan compares the current scan to the latest saved scan in the same origin + pathname scope. Query strings and hash fragments are ignored for matching.

How do I get the beta package?

Approved testers use the current release package route documented on Installation (GitHub Releases/private approved package path for this beta).

Where do I report bugs?

Use GitHub Issues. Remove sensitive URLs, selectors, and snippets before posting public reports.

What should I avoid scanning?

Do not scan pages or environments you are not authorized to inspect. Do not publish sensitive URLs, selectors, snippets, credentials, or confidential content in public issue reports.

What is stored locally and for how long?

Settings, history, workflow notes, and optional review state are stored in extension local storage until the user clears data or removes the extension. See Privacy and Local Data and Privacy Policy.
