Support boundary

Known limitations

A11Y Cat supports accessibility review workflows. It does not replace manual testing, real assistive-technology validation, or full conformance review.

Do not claim: full WCAG compliance, complete issue coverage, or real screen-reader sign-off from the extension alone.

Manual AT evidence still required

Screen Reader Review is a virtual review aid. It is not a substitute for VoiceOver, NVDA, or JAWS testing.

Restricted pages and frames

Browser-managed pages, protected frames, and inaccessible cross-origin contexts can limit scan coverage or evidence capture.

State coverage limits

Hidden, delayed, authenticated, or interaction-dependent states are not all exercised automatically during a single scan.

File access and scope limits

  • file:// pages require explicit browser permission and still have practical limits.
  • State-limited review can miss content behind dialogs, menus, tabs, accordions, validation messages, and authenticated flows.
  • Review and advisory items are not confirmed failures until a person checks them in context.

Accessibility limitation matrix

This table separates what the extension can signal from what still requires manual test evidence.

Limitation types and required follow-up
Limitation type Why it happens What it does not mean Required follow-up
Restricted page context Browser or host security boundary Not a pass Manual inspection in allowed environment
Protected or inaccessible frame Frame access blocked by context/origin policy Not a confirmed fail by itself Inspect frame content with authorized tools
State-limited scan Single current UI state was evaluated Not complete journey coverage Test dialogs, tabs, validation, auth states manually
Engine-limited or incomplete result Insufficient runtime/context for safe verdict Not a pass and not an automatic fail Manual judgment with page context
Virtual screen reader review Modelled review aid, not assistive tech runtime Not AT sign-off Test with VoiceOver, NVDA, or JAWS

Language and spelling limits

  • Spelling support is currently English-only in the bundled beta dictionary set.
  • Unsupported languages are skipped rather than being checked as English.
  • Unknown or missing language information can limit what the language and spelling checks can evaluate safely.

What to do when a limitation appears

  1. Read the limitation text instead of assuming the page passed.
  2. Review protected or stateful parts of the experience manually.
  3. Use diagnostics export if the limitation may be caused by runtime boundaries.
  4. Verify with real assistive technology where user impact is likely.
