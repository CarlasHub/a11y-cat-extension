Public documentation

Support and contact

Use these routes for product support, privacy-safe issue reporting, and Chrome Web Store contact preparation.

Support routes

NeedRouteNotes
Bug report or product feedbackGitHub IssuesUse safe demo pages or redact sensitive URLs, selectors, screenshots, and exported evidence before sharing.
Documentation issueGitHub IssuesInclude the page URL and the outdated or unclear text.

What to include

  • Extension version and browser version.
  • Whether the page was a normal web page, restricted page, iframe-heavy page, or local file.
  • Which workflow was used: scan results, dashboard, scanner module, Screen Reader Review, Browser Preference QA Controls, export, or local data clearing.
  • Relevant diagnostics text or exported support JSON only after sensitive data has been reviewed.
  • Expected result and actual result.

Privacy-safe support

Exports can contain page URLs, selectors, snippets, screenshots, reviewer notes, and diagnostics. Do not attach private client pages, credentials, personal data, or internal QA evidence to public issues.
