Installation

Install the current extension build

The documented install path is the Chrome or Chromium MV3 extension. The public site does not claim general-availability production rollout.

Current boundary: install and test the extension as a controlled beta build unless you have a separate approved release channel.

Recommended: Private beta install

Private beta packages are distributed to approved testers only. If you are part of the beta, use the package link or direct package handoff provided by the maintainer.

Current beta distribution channels
ChannelStatusHow access works
Chrome Web Store private listingNot used in this releaseNo private store listing is currently documented for v1.0.0-beta.
GitHub Releases ZIPPrimary beta channelApproved testers download the signed beta package from GitHub Releases.
Direct approved beta packageFallback channelIf the release asset is unavailable, request the approved package from the maintainer through GitHub Issues.

Current beta package identity: a11y-cat-mv3-v1.0.0.zip (extension version 1.0.0).

  1. Confirm you are an approved beta tester.
  2. Open GitHub Releases and download a11y-cat-mv3-v1.0.0.zip.
  3. If that file is not available, stop and request the approved beta package from the maintainer; do not switch to a developer source build as a substitute beta package.
  4. Unzip the package locally.
  5. Open chrome://extensions, enable Developer mode, click Load unpacked, and select the unzipped package folder.

Verify the installed version

  1. In chrome://extensions, open the A11Y Cat extension card and confirm Version 1.0.0 is displayed.
  2. Confirm the installed package filename is a11y-cat-mv3-v1.0.0.zip.
  3. Check the included package metadata file (a11y-cat-mv3-v1.0.0.zip.metadata.json) for the exact build commit when you need build-level traceability.

Report installation issues through GitHub Issues with extension version, browser version, operating system, and the failing step.

What permissions you will see

activeTab

Lets the runtime inject into the current tab only after the user clicks the toolbar action.

scripting

Runs the packaged extension runtime inside the active page after explicit user action.

storage

Stores local settings, local issue state, history, review state, and related workflow data.

The public release build does not declare broad host_permissions, hosted-service permissions, analytics permissions, or remote code execution permissions.

Developer build from source

Developer build only. This path is for contributors and internal diagnostics, not for standard beta tester rollout.

npm install
npm run build:extension
npm run check:extension
  1. Open chrome://extensions.
  2. Enable Developer mode.
  3. Select Load unpacked.
  4. Choose the built dist-extension/ directory.

Developer verification commands:

npm run build:extension:test
npm run check:extension:test
npm run test:extension:smoke
  • npm run build:extension:test: builds the extension in test mode; passing output completes without build errors.
  • npm run check:extension:test: runs extension package checks for test configuration; passing output reports no gate failures.
  • npm run test:extension:smoke: runs smoke tests for extension startup and core routes; passing output reports all smoke checks passing.

What to check after loading

  • The toolbar icon opens the extension panel on a supported page.
  • The panel renders Dashboard, Scan Results, and the documented review modules.
  • Scan, filters, exports, and local data controls are visible.

Troubleshooting installation

  • If the extension does not appear in chrome://extensions, reload the unpacked folder and verify Developer mode is enabled.
  • If the panel opens but scan actions are unavailable, refresh the target page and retry from a supported https:// page context.
  • If a build folder is rejected, rerun build commands and confirm the selected directory contains the generated extension manifest.

For deeper runtime issues, open Troubleshooting and include extension version, browser version, and OS in the report.

Known install limitations

  • Private beta distribution is controlled; testers without approved access cannot install from a public store listing in this release.
  • Restricted pages such as chrome:// and other protected browser contexts do not support panel injection.
  • The extension package is Chromium-focused in this release; Firefox and Safari packages are not shipped here.
