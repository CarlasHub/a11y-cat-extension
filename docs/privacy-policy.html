Policy

Privacy policy

A11Y Cat processes scan data locally in the browser runtime. This policy covers what is stored locally, what can be exported, and what is not sent to developer infrastructure.

Effective date: 1 May 2026

Controller and contact

  • Maintainer: Carla Gonçalves
  • Privacy and support route: GitHub Issues
  • Project repository: CarlasHub/a11y-cat-extension
  • Release identity: A11Y Cat Extension Beta · Version v1.0.0-beta · Build 4b05134 · Docs updated 2026-05-01

Data processing summary

  • Scan analysis runs locally in your browser profile.
  • Findings, notes, and workflow state are stored locally via extension storage.
  • Exports are created only on explicit user action.
  • The public package does not include analytics or telemetry.
  • The public package does not require a developer backend or database.

Legal basis and scope (GDPR / UK GDPR)

This policy describes the current public beta package. Earlier internal development builds may have used experimental keys or test-only settings that are not part of the current public beta.

This extension is a local tool. Personal data handling is user-controlled on the local device. Where GDPR or UK GDPR applies, processing is limited to user-initiated local inspection and optional user-initiated export. The maintainer does not receive scan data automatically from the public package.

This documentation is not legal advice. Organisations using A11Y Cat should review whether their own use of exports, screenshots, issue evidence, or page content is compatible with their internal data protection policies.

Historical key note

Earlier internal development builds used storage keys that are not present in the current public beta package. If you previously installed an internal build, clearing all extension data from the extension UI removes residual keys.

Plain-language permissions explanation

  • activeTab: lets the extension inspect the current tab only after you click the extension action.
  • scripting: injects the packaged scanner runtime into the active page context.
  • storage: stores local settings, history, notes, and workflow state on your device.

Exports and sharing risk

Exports can include selectors, snippets, page URL, and page title. Review export files before sharing outside your team. After a file is downloaded or shared, handling of that file is controlled by the user and their organisation.

Network requests during scanning

The Broken Links review sends same-origin HEAD or GET requests to verify destination health. These requests originate from the page context, not from A11Y Cat infrastructure. Cross-origin links are not fetched and remain unverified. No request data is sent to developer-controlled servers by the public package.

How to clear data

Use Scan Results → Privacy and data use → Clear all local extension data, or remove the extension to clear extension-managed storage.

What this build does not do

  • It does not upload scan results to a developer backend by default.
  • It does not include analytics or telemetry endpoints in the public release build.
  • It does not claim automated compliance certification.
