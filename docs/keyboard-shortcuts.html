Keyboard shortcuts and panel interaction

Keyboard behavior for extension entry, panel interaction, and known shortcut limitations in this release.

Keyboard shortcut reference
ActionDefault shortcutHow to configureNotes
Open extension actionNo default assigned in this releaseOpen chrome://extensions/shortcuts and assign a shortcut to A11Y CatBrowser-managed extension action shortcut.
In-panel custom shortcutsNoneNot configurable in this releaseThe extension panel does not currently provide dedicated keyboard shortcut commands.

Panel tab order

  1. Panel header controls (theme/settings/panel controls)
  2. Scan actions and primary result controls
  3. Result lists and issue actions (highlight/ticket/export)
  4. Diagnostics and privacy/local-data controls

Focus management behavior

  • The panel does not trap keyboard focus in this release.
  • Focus remains in normal DOM order across visible controls.
  • Focus restoration to prior page element is browser-managed after panel close.
Known limitation: no dedicated in-panel keyboard shortcut set is shipped in this release.
