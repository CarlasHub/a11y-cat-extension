Keyboard and interaction notes

Keyboard expectations for the documentation site and the extension review workflow.

Documentation site keyboard behaviour

AreaExpected behaviourNotes
Skip linkMoves focus to main content.Visible on focus.
Top navigationLinks are reachable in source order.Visible focus ring remains enabled.
Side navigationGrouped links are reachable without trapping focus.On smaller screens the navigation scrolls independently and does not hide content.
TablesTables scroll horizontally when needed.Headers use semantic table cells.
FAQ detailsSummary elements toggle with keyboard.Native details/summary semantics are used.

Extension review interaction expectations

  • Use keyboard-only operation as part of manual review; the extension does not prove keyboard accessibility by itself.
  • When reviewing a page, test focus order, skip links, controls, dialogs, menus, validation, zoom/reflow, and screen reader behaviour with the real product state.
  • Use A11Y Cat evidence to plan and record checks, not to replace them.
