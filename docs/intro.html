What A11Y Cat is

A11Y Cat is a Chrome MV3 extension for local-first accessibility inspection on the current page. It combines deterministic scan output with structured manual-review buckets, local history, and local evidence export.

Source of truth: the Extension UI feature guide is the authoritative user-facing reference for visible labels, result categories, counts, exports, storage behaviour, and release limitations.

Who it is for

Accessibility engineers

Use it to inspect live-page issues, review selectors and evidence, and separate deterministic findings from uncertain review items.

QA and release reviewers

Use it to collect local evidence, track limitations honestly, and export handoff material without claiming unsupported certainty.

Product and design teams

Use it to understand the extension’s issue categories, counts, and review prompts before making remediation decisions.

What it helps with

  • Deterministic scan results with confirmed issues, explicit review buckets, severity counts, and limitation surfaces.
  • Focused review modules for headings, language mismatch, spelling, alt text, metadata, broken links, and text scale/reflow review.
  • Local workflow support including history, comparison, issue-state export/import, diagnostics, and screen-reader review artifacts.

What it does not prove

  • full WCAG compliance
  • that manual accessibility testing is unnecessary
  • that real screen reader testing is unnecessary
  • that every interaction state was tested automatically
  • that restricted pages or protected frames were fully inspected
  • that all accessibility issues were found
Do not claim: A11Y Cat is not a compliance certificate and the virtual Screen Reader Review surface is not real NVDA, JAWS, or VoiceOver sign-off.

How to start using it

  1. Install the unpacked extension or the current agreed beta build.
  2. Open the extension from the browser toolbar on a supported page.
  3. Run the deterministic scan.
  4. Review Confirmed Issues first, then review buckets, limitations, and exports.
  5. Use the UI feature guide whenever a visible label, count, or action needs exact explanation.
