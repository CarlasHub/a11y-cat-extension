What A11Y Cat is

A11Y Cat is a Chrome MV3 extension for local-first accessibility inspection on the current page. It combines deterministic live-page checks with structured review workflows, local history, and local evidence export.

Who it is for

  • Accessibility engineers reviewing live page issues in the browser
  • QA and release reviewers who need local evidence exports and honest limitation reporting
  • Product and design teams who need a structured way to work through confirmed issues, review items, and limitations

What it helps with

  • Deterministic WCAG scan results on the current page state
  • Manual-review buckets such as visual composition, human judgement, engine-limited, and state-limited review
  • Headings, language, spelling, alt text, metadata, broken links, and Screen Reader Review workflows
  • Local exports for CSV issue logs, JSON evidence bundles, diagnostics, and review handoff artifacts

What it does not prove

  • full WCAG compliance
  • that manual accessibility testing is unnecessary
  • that real screen reader testing is unnecessary
  • that every interaction state was tested automatically
  • that restricted pages or protected frames were fully inspected
  • that all accessibility issues were found

How to start using it

  1. Install the unpacked extension or the current beta package.
  2. Open the extension from the browser toolbar on a supported page.
  3. Run the deterministic scan.
  4. Review Confirmed Issues first, then review buckets and scan limitations.
  5. Use the UI guide when a label, count, export, or workflow control is unclear.
Source of truth: the extension UI feature guide remains the authoritative user-facing reference for visible UI labels, scanner taxonomy, result categories, exports, local storage behaviour, and release limitations.
