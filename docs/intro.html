What A11Y Cat is

A11Y Cat is a local-first accessibility inspection extension for developers, QA teams, and accessibility reviewers. It combines deterministic scan output with review buckets, local history, local exports, and explicit limitation reporting.

Source of truth: the Extension UI feature guide is the authoritative user-facing reference for visible labels, result categories, counts, exports, storage behaviour, and release limitations.

Who it is for

Developers

Use confirmed findings, preserved rule provenance, selectors, and exports to fix issues in the current tested state.

QA teams

Use review buckets, diagnostics, and local history to triage regressions and document unresolved states.

Accessibility reviewers

Use the feature guide, limitations, and focused review modules to decide what still needs manual validation.

Product stakeholders

Use the documentation to understand the product boundary, the confidence level of results, and the limits of automation.

What it helps with

  • Confirmed automated findings with source attribution and preserved engine metadata.
  • Review-only output for incomplete, visual, contextual, and state-limited issues.
  • Focused review modules for headings, language, spelling, alt text, metadata, broken links, and text scaling.
  • Local workflow support including history, comparison, issue-state export/import, diagnostics, and virtual review logs.

What it does not prove

  • full WCAG compliance
  • that manual accessibility testing is unnecessary
  • that real screen reader testing is unnecessary
  • that every interaction state was tested automatically
  • that restricted pages or protected frames were fully inspected
  • that all accessibility issues were found
Private beta builds are intended for controlled testing. They are not public production sign-off.

How to start using it

  1. Install the unpacked extension or the current agreed beta build.
  2. Open the extension from the browser toolbar on a supported page.
  3. Run the deterministic scan.
  4. Review Confirmed Issues first, then review buckets, limitations, and exports.
  5. Use the UI feature guide whenever a visible label, count, or action needs an exact explanation.
